You are building a lightweight AI engineering agent toolkit.

Create a reusable repository structure for AI agent runners.

Goals:
- Keep it minimal.
- Reliability over complexity.
- Shell-based runners.
- Prompts separated from execution logic.
- No framework.
- Easy to understand and modify.

Architecture:

Create:

agents/
- runner.sh

prompts/
- issue-worker.md
- code-review.md
- test-writer.md
- bug-hunter.md
- refactor.md
- security-review.md
- release.md

Design principles:

1. The runner is generic.
2. Prompts define the agent behavior.
3. Runner responsibilities:
   - validate environment
   - check dependencies
   - verify prompt exists
   - ensure git state is safe
   - create isolated branch
   - execute AI agent
   - detect completion/failure signals
   - summarize results

4. Do not:
   - create a complex framework
   - add unnecessary dependencies
   - add databases
   - add configuration systems unless required

Runner requirements:

- Use bash.
- Use strict mode.
- Fail fast.
- Use clear sections.
- Keep variables near the top.
- Support selecting prompts through a variable.
- Use signal files:
  - .agent-complete
  - .agent-stuck

Prompt requirements:

Every prompt should follow this style:

# Agent Name

## Goal

Short goal.

## Instructions

Numbered workflow.

If complete:
- create .agent-complete

If blocked:
- create .agent-stuck with reason

Keep prompts concise.

Agent roles:

Issue Worker:
- fetch issues
- pick one
- implement
- test
- commit

Code Review:
- inspect git diff
- review changed files
- report findings
- do not modify code

Test Writer:
- identify missing tests
- add coverage
- run tests

Bug Hunter:
- inspect code for bugs
- report findings

Refactor:
- improve structure
- preserve behavior
- run tests

Security Review:
- inspect vulnerabilities
- report risks

Release:
- prepare release notes
- verify readiness

Generate:
- folder structure
- runner.sh
- all prompt files
- README.md
- Mermaid architecture diagram

Prefer boring, readable engineering over clever solutions.

## Design Skills Goals

- Be able to design systems in 30–45 minutes
- Break down requirements clearly (functional + non-functional)
- Choose appropriate architecture under constraints
- Justify tradeoffs (latency vs consistency vs cost)
- Identify bottlenecks and propose scaling solutions

---

## Evaluation Metrics

- Can I clearly define system requirements?
- Can I explain architecture in a structured flow?
- Can I justify each major design decision?
- Can I identify bottlenecks and failure points?
- Can I propose scalable improvements?
- Can I reason about tradeoffs (not just list solutions)?

---

## Study Approach

1. Understand problem requirements (functional + non-functional)
2. Define high-level architecture first
3. Break into components (API, DB, cache, queue, etc.)
4. Identify scaling bottlenecks
5. Introduce improvements step-by-step
6. Compare alternative designs and tradeoffs
7. Revisit and refine design after learning gaps
